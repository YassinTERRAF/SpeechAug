# Augmentation Protocol

This document describes the acoustic augmentation protocol implemented in **SpeechAug**.

SpeechAug is a task-independent augmentation toolkit for generating controlled noisy, reverberant, and noisy-reverberant speech conditions. The protocol is designed for speech processing experiments such as speaker identification, speaker verification, overlapping speech detection, ASR, diarization, VAD, speech emotion recognition, and speech enhancement evaluation.

---

## 1. Goal

The goal of the protocol is to generate realistic acoustic variants of clean speech recordings while preserving the original relative file paths and, by default, the original signal duration.

Given a clean speech file:

```text
x[n]
```

SpeechAug can generate:

```text
noisy speech
reverberant speech
noisy-reverberant speech
```

The tool modifies only the audio waveform. It does not modify task labels or annotations.

---

## 2. Input Speech

The input is a clean speech dataset organized under a root directory:

```text
clean_root/
└── relative/path/to/file.wav
```

A manifest specifies the relative paths:

```text
train speaker001/utt001.wav
valid speaker001/utt002.wav
test speaker002/utt001.wav
```

The clean file is loaded as:

```text
clean_root / relative_path
```

The split field is used to select the corresponding train or test noise split.

---

## 3. Dataset Split and Noise Split Mapping

SpeechAug maps dataset splits to noise splits as follows:

```text
train -> train noise
valid -> train noise
dev   -> train noise
val   -> train noise
test  -> test noise
eval  -> test noise
all   -> train noise
```

This allows training and validation data to use one set of noise sources, while test or evaluation data uses unseen noise sources.

This is useful for measuring generalization under unseen acoustic interference.

---

## 4. Supported Acoustic Conditions

SpeechAug supports three main acoustic conditions.

---

## 4.1 Noisy Speech

Noisy speech is generated by adding a scaled noise signal to clean speech.

```text
y[n] = x[n] + alpha * v[n]
```

where:

```text
x[n] = clean speech
v[n] = selected noise signal
alpha = scaling factor for target SNR
y[n] = noisy output speech
```

Supported noise types are:

```text
babble
music
noise
awgn
```

The non-AWGN noise types are loaded from a MUSAN-derived noise directory. AWGN is generated synthetically.

---

## 4.2 Reverberant Speech

Reverberant speech is generated by convolving the clean speech with a room impulse response.

```text
y[n] = x[n] * h_s[n]
```

where:

```text
x[n]   = clean speech
h_s[n] = speech RIR
*      = convolution
y[n]   = reverberant output speech
```

The default room categories are:

```text
smallroom
mediumroom
largeroom
```

The output is kept at the same duration as the clean input by default.

---

## 4.3 Noisy-Reverberant Speech

Noisy-reverberant speech combines reverberated speech with reverberated environmental noise.

```text
x_r[n] = x[n] * h_s[n]
v_r[n] = v[n] * h_v[n]
y[n]   = x_r[n] + alpha * v_r[n]
```

where:

```text
x[n]   = clean speech
v[n]   = selected environmental noise
h_s[n] = speech RIR
h_v[n] = noise RIR
x_r[n] = reverberated speech
v_r[n] = reverberated noise
alpha  = scaling factor for target SNR
y[n]   = noisy-reverberant output speech
```

The speech RIR and noise RIR are selected from the same room category. They can be different RIR files.

For AWGN, the protocol is different:

```text
x_r[n] = x[n] * h_s[n]
y[n]   = x_r[n] + alpha * w[n]
```

where:

```text
w[n] = synthetic Gaussian white noise
```

AWGN is added directly and is not reverberated.

---

## 5. SNR Scaling

Noise is scaled to achieve a target signal-to-noise ratio.

For noisy speech, the reference signal is the clean speech:

```text
SNR = 10 log10( P_clean / P_noise_scaled )
```

For noisy-reverberant speech, the reference signal is the reverberated speech:

```text
SNR = 10 log10( P_reverberant_speech / P_noise_scaled )
```

The power of a signal is computed as:

```text
P_x = mean( x[n]^2 )
```

For a target SNR in dB, the scaling factor is:

```text
alpha = sqrt( P_reference / (P_noise * 10^(SNR_dB / 10)) )
```

Then:

```text
scaled_noise = alpha * noise
```

Default SNR levels are:

```text
-5, 0, 5, 10, 15, 20 dB
```

---

## 6. Noise Duration Handling

Noise clips may have different durations. Some MUSAN noise or music files may be a few seconds long, while others may be several minutes long.

SpeechAug automatically matches the noise duration to the clean utterance duration.

If the selected noise is longer than the clean utterance:

```text
random segment is cropped
```

If the selected noise is shorter than the clean utterance:

```text
noise is repeated and then cropped
```

If the selected noise has the same length:

```text
noise is used directly
```

This ensures that the noise signal and speech signal have the same duration before SNR scaling.

---

## 7. Babble Generation

Babble noise is generated from the speech portion of MUSAN.

During MUSAN preparation, speech clips are split into disjoint train and test source pools. Babble files are then generated by mixing multiple speech clips:

```text
babble = speech_1 + speech_2 + ... + speech_K
```

The default number of mixed speech sources is:

```text
3 to 7 speakers
```

The default babble duration can be controlled with:

```bash
--babble-duration
```

For example:

```bash
--babble-duration 10.0
```

generates 10-second babble files.

---

## 8. RIR Selection

RIRs are loaded from a directory structure such as:

```text
rir_root/
└── simulated_rirs/
    ├── smallroom/
    ├── mediumroom/
    └── largeroom/
```

For reverberant speech, one RIR is selected for the clean speech.

For noisy-reverberant speech, two RIRs are selected from the same room category:

```text
speech RIR -> applied to clean speech
noise RIR  -> applied to environmental noise
```

If only one RIR is available in a room category, it may be reused for both speech and noise.

---

## 9. RIR Alignment

SpeechAug can align the RIR near the estimated direct path before convolution. This avoids adding artificial leading silence caused by the initial delay of the RIR recording.

The alignment is controlled by:

```yaml
direct_threshold_fraction: 0.2
preroll_ms: 1.0
```

This keeps a small amount of signal before the estimated direct path.

---

## 10. Output Length

By default, SpeechAug keeps the output duration equal to the clean input duration:

```yaml
keep_length: true
```

This is useful for tasks with time-aligned labels, such as frame-level overlapping speech detection, VAD, diarization, and ASR.

If the full convolution tail is needed, the user may disable length preservation if supported by the configuration.

---

## 11. Clipping Protection

After mixing or convolution, the signal is protected from clipping. If the peak amplitude exceeds the allowed maximum, the waveform is scaled down.

This prevents invalid clipped output files while preserving the relative acoustic condition.

---

## 12. Reproducibility

SpeechAug uses deterministic seeds based on the utterance path and augmentation condition.

The random seed depends on information such as:

```text
condition
split
relative path
noise type
SNR
room type
global seed
```

This makes noise selection, RIR selection, and random cropping reproducible across runs when the same inputs and configuration are used.

---

## 13. Output Organization

The original relative file path is preserved under each output condition.

Example output structure:

```text
output_root/
├── noisy/
│   ├── babble/
│   │   ├── -5dB/
│   │   ├── 0dB/
│   │   └── ...
│   ├── music/
│   ├── noise/
│   └── awgn/
├── reverb/
│   ├── smallroom/
│   ├── mediumroom/
│   └── largeroom/
└── noisy_reverb/
    ├── smallroom/
    │   ├── babble/
    │   ├── music/
    │   ├── noise/
    │   └── awgn/
    ├── mediumroom/
    └── largeroom/
```

Example generated files:

```text
output_root/noisy/babble/0dB/s1/example.wav
output_root/reverb/smallroom/s1/example.wav
output_root/noisy_reverb/smallroom/music/10dB/s1/example.wav
```

---

## 14. Notes for Downstream Tasks

SpeechAug is task-independent. It augments audio only.

For speaker identification and speaker verification:

```text
speaker labels remain unchanged
```

For frame-level overlapping speech detection:

```text
time-aligned labels can remain compatible when output length is preserved
```

For ASR, diarization, VAD, and other time-sensitive tasks:

```text
users should verify that labels remain valid for their setup
```

SpeechAug does not generate or modify labels.

---

## 15. Summary

SpeechAug implements a controlled acoustic augmentation protocol with:

```text
additive noise
room reverberation
combined noisy-reverberant conditions
synthetic AWGN
MUSAN-derived babble/music/noise
RIRS_NOISES-derived room impulse responses
train/test noise separation
reproducible noise and RIR selection
duration-preserving output by default
```
